## What Was Implemented
Replaced the minimal Components page stub with a full Server Component implementation that supports three view modes (Grid, List, Coverage) accessible via URL-based tab navigation. All filtering and sorting is handled via `<Link>` elements that update searchParams, requiring zero client-side JavaScript state. The page gracefully handles a missing `components.json` file with a descriptive empty state.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Navigation mechanism | URL searchParams via `<Link>` | Server Component constraint — no useState; all state lives in URL |
| hasStory / hasTests derivation | `status === 'done'` proxy | No story/test fields in ComponentSummary schema; avoids schema change |
| Layer derivation | Keyword matching mirrored from data.ts | PRIMITIVE_KEYWORDS / DOMAIN_KEYWORDS not exported; duplicated per plan |
| Sort implementation | `Array#toSorted()` | ESLint `unicorn/no-array-sort` rule enforced in this project |
| SortHeader placement | Module-level named function (not nested) | Avoids potential issues; ESLint `unicorn/prefer-switch` triggered by nested if/else-if |
