## What Was Implemented
Created `ArchitectureTabs.tsx` as a `'use client'` component with three tabs: ADR Bible (renders the existing `ADRViewer` component), Grand Workplan (renders the markdown file via `ReactMarkdown + remark-gfm`, or an empty-state with the missing file path), and Tech Stack (2-col grid of layer cards — Frontend, Data, Services, Infrastructure). Rewrote `page.tsx` as a server component that loads all ADR content via `getADRList` + `getADRContent`, reads the Grand Workplan markdown file with try/catch fallback to null, defines the hardcoded `TECH_STACK_LAYERS` constant, and passes everything to `ArchitectureTabs`.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Tab state location | `ArchitectureTabs.tsx` client component | Server component can't hold interactive state; client child keeps server data-loading clean |
| TechStackLayer type | Local interface in ArchitectureTabs.tsx, re-exported | Avoids external dependency; page.tsx imports the type for the typed constant |
| Negated condition | `workplanMarkdown === null` first (empty state) | `unicorn/no-negated-condition` lint rule — positive condition must come first |
| `let workplanMarkdown` | Declared without initializer | `no-useless-assignment` — every branch assigns before use, initial `= null` was redundant |
| ADR body fallback | `{ ...meta, body: '' }` on null result | Prevents dropping ADRs from the list if a single file fails to parse |
