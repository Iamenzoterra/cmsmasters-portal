**Root tsconfig.json paths:**

- NO `@cmsmasters/*` path alias at root level

**CC tsconfig.json paths:**
```json
"paths": {
  "@/*": ["./*"],
  "tailwindcss": ["./types/tailwindcss.d.ts"]
}
```
- `@/*` maps to CC-internal files — standard Next.js pattern
- NO `@cmsmasters/ui` alias

**Symlink check:** `node_modules/@cmsmasters/` does NOT exist
- npm workspaces only creates symlinks for packages that are depended on
- Since no app lists `@cmsmasters/ui` as a dependency, no symlink is created
- `require.resolve('@cmsmasters/ui')` FAILS

**Resolve test:**
```
DOES NOT RESOLVE: Cannot find module '...\node_modules\@cmsmasters\ui\index.ts'
```
Note: Node attempted to resolve at `node_modules/@cmsmasters/ui/index.ts` — npm knows about the package from workspaces config, but the directory isn't hoisted because it's not a dependency of anything.

**Conclusion:** To make `import { Button } from '@cmsmasters/ui'` work in CC:
1. Add `"@cmsmasters/ui": "*"` to CC devDependencies (**NOT `"workspace:*"` — that's pnpm syntax, we use npm**)
2. Run `npm install` — npm will create the symlink
3. `packages/ui/index.ts` must exist (currently missing)
4. No tsconfig path alias needed — npm workspace resolution handles it via node_modules symlink
