# Execution Log: CMS-371 — Fix: Content section shows themes and docs and blog post counts on one line

> Duration: 5 minutes

## What Was Implemented

Added `type?: 'blog' | 'doc'` to the `ContentStatus` interface in `lib/types.ts`. Updated `scan.ts` to tag each content entry during the `scanContent()` loop — entries whose `app` equals `'blog'` or whose tasks have `group === 'blog'` are tagged `'blog'`; all others are tagged `'doc'`. Updated `report.ts` to read the `type` field from content entries, compute `blogCount` dynamically via filter, restrict `docsCount` to non-blog approved entries, and split the single CONTENT line into four separate lines (header + one line per metric).

## Key Decisions

| Decision | Chosen | Why |
|----------|--------|-----|
| Where to tag content type | `scan.ts` (producer) | Keeps type logic in the scanner; report.ts stays a pure consumer |
| Blog detection criteria | `app === 'blog'` OR `group?.toLowerCase() === 'blog'` | Covers both current and future data shapes |
| docsCount filter | `type !== 'blog' && status === 'approved'` | Only counts approved non-blog entries, matching original intent |
| CONTENT format | 4 separate lines with aligned labels | Fixes the one-liner display issue, readable in Slack/email copy-paste |
