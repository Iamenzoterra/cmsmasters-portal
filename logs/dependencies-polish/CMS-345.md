## What Was Implemented
Created `DependencyGraph.tsx` with two tab-switchable views: Package Dependencies and Phase Dependencies. The package view renders an SVG graph with 6 packages on the left connected by polylines to 7 apps on the right; clicking any node highlights its edges and shows a tooltip. The phase view renders a Gantt-like SVG timeline with bars sized by `estimatedWeeks`, colored by status, a full-width Content track, and arrow markers between consecutive phase bars. All color values are derived from `tokens.ts` via module-level `COLOR_*` constants with no inline hex strings.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Helper functions scope | Module-level | `unicorn/consistent-function-scoping` lint rule requires moving pure functions out of components |
| `getStatusDotFill` vs `getPhaseStroke` | Merged into `getPhaseStroke` | Both were identical; `sonarjs/no-identical-functions` error; dot color equals stroke color for all statuses |
| Nested ternary | Replaced with `if` block | `unicorn/no-nested-ternary` and `sonarjs/no-nested-conditional` lint errors |
| Edge midpoint routing | Two right-angle waypoints at MID_X=450 | Creates clean orthogonal routing between left and right columns |
| Phase SVG height | 180px | Fits bar (44px at y=40) + weeks label + content track (28px at y=110) comfortably |

## Issues & Workarounds
- **`unicorn/consistent-function-scoping`**: Four helper functions (`getPhaseFill`, `getPhaseStroke`, `getPhaseFillOpacity`, `getStatusDotFill`) were initially inside the component body. Moved to module scope to satisfy lint rule.
- **`sonarjs/no-identical-functions`**: `getStatusDotFill` was identical to `getPhaseStroke`. Removed `getStatusDotFill` and used `getPhaseStroke` for dot color as well.
- **`unicorn/no-nested-ternary`**: Replaced ternary chain for `nameLabel` with an `if` block.
