## What Was Implemented
Created a new `GlobalSearch` client component that opens a search overlay on Cmd+K, filters across pre-loaded search entries, and navigates on selection. Updated `layout.tsx` to be an async Server Component that builds a normalized search index from `getPhases`, `getComponents`, `getContentStatusEntries`, and `getADRList`, then passes it into `GlobalSearch` as props. Added an inline `<Script strategy="afterInteractive">` block for number-key (1–6) navigation shortcuts with guards against input/textarea/contentEditable focus.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Keyboard shortcuts delivery | Inline `<Script strategy="afterInteractive">` | Executes once per full page load in App Router — no duplicate listener risk, no client component needed |
| `window` vs `globalThis` | `globalThis` | ESLint `unicorn/prefer-global-this` rule enforced in this project |
| Components JSON shape | `Array.isArray` + nested-object fallback | `components.json` may be `{ components: [...] }` — same pattern used by `app/components/page.tsx` |
| Search index scope | Pages, phases, tasks, components, themes, ADRs | All pages' primary data sources, normalized to `SearchEntry` |
