## What Was Implemented
Created `GlobalSearch.tsx` as a `'use client'` command palette component that accepts a `searchIndex` prop from a Server Component parent. The component opens on Cmd+K or `/` (when not focused in an input/textarea/contenteditable), closes on Escape or backdrop click, auto-focuses the search input when opened, filters results client-side grouped by type (Tasks, Components, Themes, ADRs), caps each group at 5 results, and navigates via `useRouter` on item click.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Data source | `searchIndex` prop | Follows CC convention: data loads server-side, component is UI-only |
| State management | useState (isOpen, query) | No external state lib needed; component is self-contained |
| Keydown listener | document-level useEffect | Global hotkey requires document-level listener |
| Result grouping | useMemo + Map keyed by TYPE_ORDER | Deterministic ordering and O(n) filtering |
| No-results | Shown only when query.trim() !== '' | Avoids false "no results" on initial open |
