### `infra-tooling`
- **Invariant — Port allocation:** layout-maker=7700/7701, block-forge=7702 (strictPort), studio-mockups=7777. Phase 0 verifies **7703** is free.
- **Trap — `tools/*` is NOT an npm workspace.** Root `package.json` `workspaces` covers `apps/*`, `packages/*`, `tools/layout-maker` only (NOT `tools/block-forge`). CONVENTIONS.md §0 codifies the pattern: **`cd tools/<X> && npm <cmd>`** for package-scoped commands. **WP-030 §1.7's draft `--workspace=tools/responsive-tokens-editor` plan is WRONG** — Phase 1 must use the `cd tools/X && npm run dev` pattern (root alias matching `block-forge:` line in `package.json:24`).
- **Trap — Install dance for workspace `*` deps.** `tools/block-forge/package.json` declares `@cmsmasters/*: "*"` workspace deps; `npm install` queries the public registry and 404s. Workaround: temporarily comment out `@cmsmasters/*` lines, `npm install`, restore. Same pattern will apply to `tools/responsive-tokens-editor/`.
- **Reference — block-forge install dance** documented in `infra-tooling/SKILL.md` Traps & Gotchas.

### `pkg-ui`
- **Invariant — `tokens.css` is auto-generated by `/sync-tokens`** (Figma source). DO NOT hand-edit. HSL stored without wrapper (`228 54% 20%` not `hsl(...)`). 222 lines.
- **Invariant — `tokens.responsive.css` is hand-maintained / tool-generated.** `/sync-tokens` does NOT touch it. WP-024 left it as a 2-token scaffold; WP-030 populates it.
- **Invariant — No build step in `packages/ui/`.** Consumers import TypeScript / CSS directly via `?raw` (Vite) or `@import`. No compiled `dist/`.
- **Trap — Tailwind v4 font-size hint:** `text-[length:var(--text-sm-font-size)]` NOT `text-[var(...)]` (otherwise Tailwind reads as color).
- **Trap — Tailwind v4 bare var for sizing:** `h-[--button-height-sm]` NOT `h-[var(--button-height-sm)]` — `var()` wrapper makes Tailwind skip class generation.
- **Blast radius — changing tokens.css** affects Portal, Studio, Dashboard, Admin (everything except Command Center). Activating populated `tokens.responsive.css` → every existing block in Portal renders fluidly post-deploy.

### `studio-blocks`
- **Invariant — Studio Responsive tab `preview-assets.ts` ALREADY imports `tokens.responsive.css`** at `apps/studio/src/pages/block-editor/responsive/preview-assets.ts:19`. Currently injects empty fluid behavior (scaffold tokens unconsumed by blocks). Activating populated tokens → preview iframe shows fluid behavior automatically — no preview-assets.ts edit needed *for that side*.
- **Invariant — Path B single-wrap.** Studio's `composeSrcDoc` wraps body with ONLY `<div class="slot-inner">${html}</div>`; engine `wrapBlockHtml` provides the inner `<div data-block-shell="{slug}">`. **WP-030 must NOT touch this.**
- **PARITY contract — `apps/studio/src/pages/block-editor/responsive/PARITY.md`** cross-references `tools/block-forge/PARITY.md`. Phase 6 of WP-030 will extend the cross-reference chain to include `tools/responsive-tokens-editor/PARITY.md`.
