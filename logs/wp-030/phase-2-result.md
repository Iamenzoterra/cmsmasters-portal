## Pre-flight findings

- **checkWCAG runtime contract** (locked Phase 2 RECON, refined post-empirical-probe per `feedback_empirical_over_declarative`):
  - Signature: `({ min, max, minWidth, maxWidth }) => number[] | null` (verified via `dist/index.d.ts` L75-80)
  - Semantics: `pass = result is null OR empty array`; `fail = length-2 number array [from, to]` where numbers are viewport-width bounds of the > 2.5× ratio violation zone
  - **`null` vs `[]` distinction for pass cases is non-deterministic from inputs** — internal numeric-precision detail of utopia-core; both shapes mean "no violation". Initial PF.4 lock (declared `null = exact-edge ratio === 2.5`) was over-specified — empirical truth showed (10-25 ratio 2.5 → null) but (16-40 ratio 2.5 → []), (20-50 ratio 2.5 → null), (2-5 ratio 2.5 → []). Brain refined PF.4 + drift check at commit `6afe773b` BEFORE Phase 2 execution started.
  - Adapter `isViolation = Array.isArray(result) && result.length > 0` handles both pass shapes correctly.
- **utopia-core clamp emit format** — discovered at test execution: format is `clamp(min_rem, intercept_rem + slope_unit, max_rem)`. Unit (`vi`/`vw`) is followed by `,` (comma), NOT `)` (closing paren). Initial test regex `/vi\)/` from task.md was speculative — corrected inline to `/vi,/` to match empirical format. **Task.md verification §5/§6 grep pattern (`grep -c 'vi)' ... 'vw)'`) carries the same speculative bug** — surfaced for Brain to amend (future-phase consistency); Phase 2 verification used empirical-correct `vi,` / `vw,` patterns.
