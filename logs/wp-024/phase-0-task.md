## Context

## Domain Context (read the skills below in Phase 0 ‚Äî do not skip)

**pkg-db:**
- Key invariants: `packages/db/src/types.ts` is auto-generated by `supabase gen types`. Branded types (e.g. `BlockHooks`, `BlockMetadata`) live alongside and must be re-applied after every regen. `slot_config`, `metadata`, JSON columns are typed via these branded aliases, not raw `Json`.
- Known traps: running `supabase gen types` overwrites any hand-edits to `types.ts` ‚Äî the re-apply step is easy to miss.
- Public API: `packages/db/src/index.ts` re-exports branded types for consumers.

**pkg-validators:**
- Key invariants: validators are the **only write-path check** ‚Äî DB has no runtime validation. `createBlockSchema` / `updateBlockSchema` shape what the API and Studio accept.
- Known traps: optional JSONB fields must default to `undefined` (absent), not `{}` ‚Äî otherwise null-vs-empty-object semantics leak into the DB.
- Public API: `packages/validators/src/index.ts` exports schemas + `CreateBlockPayload` / `UpdateBlockPayload` types.

**app-portal:**
- Key invariants: block CSS is scoped under `.block-{slug}`. `resolveSlots` in `apps/portal/lib/hooks.ts` is **single-pass** ‚Äî it expects the HTML shape to be stable. Two render paths exist (`BlockRenderer` RSC and `renderBlock()` string helper) for RSC vs. slot interpolation ‚Äî they must stay in sync.
- Known traps: `stripGlobalPageRules` (in `lib/hooks.ts` or a sibling util) rewrites block CSS to remove page-level selectors; it must **not** strip `@container` rules.
- Public API: `BlockRenderer` is the canonical block-to-DOM boundary; call sites live in `app/[[...slug]]/page.tsx` and `app/themes/[slug]/page.tsx`.

**pkg-ui:**
- Key invariants: `tokens.css` is auto-generated by `/sync-tokens` ‚Äî hand-edits are forbidden. Command Center does NOT consume portal tokens. Consumers import `tokens.css` once in app globals.
- Known traps: Tailwind v4 bare-var syntax (`h-[--x]`, not `h-[var(--x)]`); HSL without wrapper (`228 54% 20%`, not `hsl(‚Ä¶)`).
- Public API: `packages/ui/src/theme/tokens.css` ‚Äî consumed by portal, dashboard, support, studio, admin globals.

**infra-tooling** (Layout Maker):
- Key invariants: PARITY-LOG discipline ‚Äî every change to `css-generator.ts` that could affect output must land with a PARITY-LOG entry AND a contract test. Layout Maker is the source of truth for layout HTML/CSS; DB is downstream.
- Known traps: `css-generator` branches on container-vs-leaf slots; the generic `.slot-inner` rule (~line 246) applies to leaf slots only. Container slots do NOT get `.slot-inner`.
- Public API: internal tooling ‚Äî no domain imports its source.
