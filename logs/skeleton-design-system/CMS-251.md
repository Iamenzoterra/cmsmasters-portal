## What Was Implemented
Created `apps/command-center/ui/Card.tsx` — a reusable Card atom component with dark theme styling. It renders children inside a padded container using `bg-surface-card`, `border-zinc-800`, `rounded-card`, and `p-card` Tailwind classes from the project's design token config. A CSS transition applies a subtle hover lift (`-translate-y-0.5`) and shadow increase (`shadow-lg shadow-black/40`) on hover. All div props including `onClick` and `className` are forwarded to the root element via `ComponentPropsWithoutRef<'div'>`.

Also created `eslint.config.mjs` (pre-existing gap — no ESLint config existed), which was required to make `npm run lint` pass.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Hover translateY | `-translate-y-0.5` (2px) | Matches spec of translateY(-2px); Tailwind 0.5 = 0.125rem ≈ 2px |
| Border color | `border-zinc-800` | Spec calls for zinc-800 border; maps to `#27272a` |
| Background | `bg-surface-card` | Custom token mapped in tailwind.config.ts to `#18181b` (zinc-900) |
| Padding | `p-card` | Custom spacing token `1.25rem` defined in tailwind.config.ts |
| Radius | `rounded-card` | Custom radius `0.875rem` defined in tailwind.config.ts |
| No `'use client'` | Server Component | No hooks or browser APIs; hover is CSS-only via Tailwind |
