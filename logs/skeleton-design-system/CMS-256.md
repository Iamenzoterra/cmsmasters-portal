## What Was Implemented
Created `apps/command-center/ui/Select.tsx` as a typed React wrapper around a native `<select>` element. The component supports either a typed `options` prop (array of `{ value, label }`) or native `<option>` children. It includes optional `label` and `error` props, forwards all standard select attributes via `forwardRef`, and renders a decorative `ChevronDown` icon positioned absolutely with `pointer-events-none`. Dark theme classes, focus ring styling, and red error state are applied using `cn()`.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| options vs children | Both supported | options prop for typed data, children as fallback for native option elements — matches plan spec |
| Chevron icon | ChevronDown from lucide-react, pointer-events-none absolute | Decorative only, does not block native select interaction |
| Label association | htmlFor + useId | Programmatic association for a11y |
| Error a11y | aria-invalid + aria-describedby | Matches Input atom pattern from CMS-255 |
