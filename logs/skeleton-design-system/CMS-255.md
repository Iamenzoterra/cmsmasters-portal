## What Was Implemented
Created `Input.tsx` as a `forwardRef`-based React input atom with dark theme styling. The component extends standard HTML input attributes with an optional `error` string prop. It renders a `zinc-900` background input with `zinc-700` border and `zinc-100` text, a focus ring using `zinc-500`, and a red border + error message below when the `error` prop is set. Accessibility is handled via `aria-invalid` and `aria-describedby` linking the input to its error message.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| `'use client'` directive | Yes | `forwardRef` with `useId` requires client rendering |
| ID generation | `useId()` with fallback to `id` prop | Ensures unique IDs for aria-describedby without requiring consumers to pass IDs |
| Error message element | `<p>` tag below input | Semantic, simple, linked via `aria-describedby` |
| Focus ring color in error state | `ring-red-500` | Consistent with error color signaling |
