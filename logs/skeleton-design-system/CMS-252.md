## What Was Implemented
Created `apps/command-center/ui/StatusBadge.tsx` as a self-contained presentational React component. It defines a `Status` union type and a `STATUS_MAP` constant mapping each status to label, Tailwind bg/text classes, and tooltip description. The component renders a non-interactive `<span>` with `rounded-badge`, `text-[11px] font-bold`, mapped color classes, a `title` attribute for tooltip, `animate-pulse` only for `in-progress`, `onClick` with `e.stopPropagation()`, and a neutral `FALLBACK_CONFIG` for unknown/missing status.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Badge element | `<span>` | Non-interactive display element; not button/link to avoid navigation |
| Tooltip mechanism | `title` attribute | Native HTML, no extra dependencies; testable via DOM |
| Pulse animation | `animate-pulse` on `in-progress` only | Matches plan spec and Tailwind built-in |
| Fallback | `FALLBACK_CONFIG` constant | Avoids runtime errors for missing/unknown status |
