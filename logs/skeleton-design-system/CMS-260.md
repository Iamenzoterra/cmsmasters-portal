## What Was Implemented
Created `apps/command-center/components/Sidebar.tsx` as a client component using `usePathname()` for active route detection. The sidebar renders a fixed-height left panel with 6 navigation items (Mission Control, Phase Tracker, Components, Content, Architecture, Dependencies), each with a Lucide icon and label. It collapses to icon-only below the xl (1280px) breakpoint via responsive Tailwind classes. The bottom section shows a last scan timestamp and a Rescan button that is disabled when no `onRescan` callback is provided.

Also updated `tailwind.config.ts` to: (1) include `./components/**` in the content scan, (2) add `accent` color token (`#3b82f6`), and (3) add `border.subtle` color token (`#1f1f23`) to support the semantic classes used in the Sidebar.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| `accent` token value | `#3b82f6` (same as `status.active`) | No dedicated accent color existed; blue is the natural accent matching the active state pattern used elsewhere |
| `border.subtle` token value | `#1f1f23` | Slightly darker than `border.default` (#27272a) to be "subtle" — used for the sidebar border-r and section dividers |
| Logo icon at collapsed size | `LayoutDashboard` icon | Compact visual identity below xl; matches Mission Control icon as the "home" identity |
| Rescan icon | `RefreshCw` | Semantically correct for a rescan/refresh action, always visible even when label is hidden |
