## What Was Implemented
Created `apps/command-center/ui/ProgressBar.tsx` as a client component. The component accepts a numeric `value` prop, clamps it to 0–100, and animates the fill width from 0 to the clamped value on mount via `useState`/`useEffect` with a 500ms ease-out CSS transition. Fill color is computed from thresholds (red <25, yellow <50, blue <75, green ≥75). An optional percentage label renders to the right in mono font. The outer wrapper exposes `role="progressbar"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, and `title` for accessibility and tooltip. An optional `onClick` prop makes the bar clickable.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Animation approach | useState(0) → useEffect sets clamped | Pure CSS transition triggered by state change on mount, no external animation library needed |
| Track height | h-1.5 (6px) | Matches plan spec; visually compact for dashboard use |
| Overflow | overflow-hidden on track | Prevents fill from visually escaping rounded corners during transition |
| Click handler | Attached only when onClick provided | Avoids cursor-pointer and click semantics on non-interactive usage |
