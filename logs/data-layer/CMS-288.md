## What Was Implemented

Created two new ADR files in `workplan/adr/`. ADR-021 defines the subscription architecture with a
three-tier model (free/pro/agency), billing lifecycle via Stripe webhooks through the Hono API
secrets boundary, entitlement rules, and a grace period policy. ADR-022 defines the Auth API and
Data Flow, covering Hono on Cloudflare Workers as the single secrets boundary, Supabase Auth with
magic link and social OAuth, a typed api-client via `hono/client` RPC in `/packages/api-client`,
per-app PKCE sessions, and shared entitlement resolvers in `/packages/auth`.

## Key Decisions

| Decision | Chosen | Why |

| Secrets boundary | Hono on CF Workers | Keeps all third-party keys out of browser bundles |
| Auth provider | Supabase Auth (magic link + social) | Avoids custom auth server, aligns with ADR-18 |
| API client | hono/client RPC typed package | Type-safe without OpenAPI code generation |
| Session isolation | Per-app PKCE | Limits blast radius of compromised session |
