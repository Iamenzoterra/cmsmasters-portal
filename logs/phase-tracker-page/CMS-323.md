## What Was Implemented
Created `TaskTable.tsx` as a reusable `'use client'` component with typed `Task` and `TaskStatus` exports. The table renders 9 columns (Status, ID, Title, Owner, App, Priority, Dependencies, Est. Hours, Act. Hours) with clickable sort headers that toggle ascending/descending order. Status cells render as colored dots using semantic design token classes. Rows fire `onSelect(task.id)` on click, and an empty state message is shown when the task array is empty.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Sort implementation | `.toSorted()` instead of `.sort()` | Required by `unicorn/no-array-sort` lint rule |
| Stable sort | Index-based tiebreaker via `.map((task, index) => ...)` | Preserves input order for equal values without library dependency |
| Nested ternary | Extracted `getSortIcon()` helper | Required by `sonarjs/no-nested-conditional` lint rule |
| Status dots | `bg-status-*` tokens + `bg-zinc-500` for todo | Uses semantic tailwind config tokens; no semantic token exists for todo |
| `cmp` initialization | Single `const` with ternary expression | Avoids `no-useless-assignment` and `unicorn/prefer-ternary` lint errors |
