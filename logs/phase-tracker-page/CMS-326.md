## What Was Implemented
Created `PhaseCard.tsx` as a `'use client'` component that renders a collapsible phase card. The card header shows the phase number badge (derived from `phase.id` via regex), title, and description. A `ProgressBar` shows task completion percentage alongside a monospace `X / Y tasks` count. Optional `estimatedWeeks` and `startDate`/`endDate` metadata appear below the progress bar. Clicking the header toggles an inline task table (ID, Title, Status, Owner, Priority columns) with `ChevronDown`/`ChevronRight` icons. Phases with `status === 'in-progress'` receive a blue ring highlight; `defaultExpanded` prop controls initial open state.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Inline task table vs reuse TaskTable | Inlined simpler table | Plan specified inline; TaskTable has incompatible local Task type and extra columns not needed here |
| Phase number extraction | `/phase-(\d+)/i` regex with fallback to `phase.id` | Handles `phase-0`, `phase-1` etc. IDs; graceful fallback for non-standard IDs |
| Task import source | `@/lib/types` | Canonical type source; avoids type mismatch with TaskTable's local re-definition |
| Priority colors | Added `critical: 'text-red-400'` | lib/types Priority includes 'critical' which TaskTable's local type omits |
