## What Was Implemented
Created `TaskFilters.tsx` as a controlled client component with a horizontal filter bar containing a search input and four Select dropdowns (Phase, Status, Owner, App). The component accepts current filter state and an onChange callback from the parent. Active non-default filters render as removable chip badges below the bar. All option arrays are derived from the `TaskStatus`, `Owner`, and `App` union types in `lib/types.ts`.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Type-driven options | `TASK_STATUSES: TaskStatus[]` + `STATUS_LABELS: Record<TaskStatus, string>` | Ensures types are used genuinely, avoids lint unused-import warnings, and keeps label mapping co-located with values |
| App count | All 10 App values from types.ts | types.ts exports 10 App values; task description said "9" but types are authoritative |
| Search icon placement | Absolute within relative wrapper div, `pl-9` on Input | Input atom wraps itself in a div, so icon is positioned via outer relative container; pl-9 shifts text to avoid overlap |
| Chip clear action | `update(key, key === 'search' ? '' : 'all')` inline ternary | Single ternary, not nested — avoids sonarjs/no-nested-conditional, keeps chip map callback concise |
