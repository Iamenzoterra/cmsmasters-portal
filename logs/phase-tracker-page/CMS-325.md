## What Was Implemented
Created `TaskDetailSheet.tsx` — a slide-in panel from the right side of the screen that displays full task details. The panel uses a CSS `translateX` transition (duration-300) controlled by whether `task` is non-null. It includes a fixed backdrop overlay for outside-click closing, Escape key handling via `useEffect`, a sticky header with title + `StatusBadge` + close button, and a scrollable body with Description, Details (owner/app/priority pills), Dependencies (clickable buttons resolving names from `allTasks`), Acceptance Criteria (CheckSquare/Square icons), Notes (pre-wrapped), and Timestamps (created/started/completed) sections.

## Key Decisions
| Decision | Chosen | Why |
|----------|--------|-----|
| Import Task type | `@/lib/types` | Full Task interface with all required fields (acceptanceCriteria, notes, completedAt, etc.) |
| startedAt field | Type assertion `task as Task & { startedAt?: string }` | Field not in Task interface but may exist at runtime; avoids widening the shared type |
| Non-null assertion avoidance | IIFE extracting startedAt to local var | ESLint rule `@typescript-eslint/no-non-null-assertion` rejects `!` operator |
| Priority colors | Hardcoded PRIORITY_STYLE record with Tailwind classes | Tailwind needs full class names at build time; cannot use dynamic interpolation |
| Acceptance criteria done state | Based on `task.status === 'done'` | No per-criterion done state in Task type; using task-level done is the correct fallback |
